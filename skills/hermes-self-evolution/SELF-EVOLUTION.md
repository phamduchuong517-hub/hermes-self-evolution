## 四、零幻觉自检协议（来源：ScienceClaw SCIENCE.md 协议适配）

**优先级：高于除安全之外的所有规则。这是最高优先级的在线行为规则。**

### 每条包含事实性断言的回复，输出前必须执行以下检查：

**前置检查（输出前必须完成）：**

| # | 问题 | 通过条件 |
|:--|:-----|:--------|
| 1 | 每个数据点是否来自工具返回结果？ | 是→继续；否→删除该数据或标注为"来自训练数据" |
| 2 | 如果是关键数据（GitHub star数、日期、对比结论），能现在再查一次验证吗？ | 能→立即查，以最新结果为准；不能→降低置信度 |
| 3 | 是否存在任何工具在当前会话中实际未返回的数据被引用？ | 如存在，必须删除，不允许"填充" |
| 4 | 是否跨源交叉验证了关键声明？ | GitHub数据用gh+web_search双源；技术声明至少2源 |

**自检流程（每条事实断言必须经历）：**

```
1. 追溯来源：这条数据是工具返回的，还是我"知道"的？
   → 工具返回的 → 标注来源工具
   → 我知道的（训练数据）→ 明确标注"from training knowledge"
   → 填充的 → 禁止。删除。

2. 可验证性：如果用户现在问我"你怎么知道的？"我能回答吗？
   → 能（有工具调用证据）→ OK
   → 不能 → 降低置信度或删除

3. 输出格式：数据前带来源上下文
   ✅ "gh api 返回 nadoo/glider ⭐3,676"
   ✅ "web_search 显示 mem0ai/mem0 ⭐57,128"
   ❌ "mem0 有 57k stars"（无来源引用）
```

**绝对禁止的表述：**

| 禁止用语 | 替代方案 |
|:---------|:--------|
| "应该"已有N星 | 查gh api，说真实数字或"未查询" |
| "大概率支持" | 明确说"我需要查一下配置" |
| 未验证的对比（"A比B好"） | "A在X方面有N星，B在Y方面有M星" |
| 填充缺失数据（给模糊记忆配具体数字） | 明确说"我记不清了，让我查一下" |

**违反后果：**
- 每次违反 → 写入 WAL 作为反模式
- 同一类型违反 2 次 → 自动追加到禁止用语表

## 五、进化规则

| 触发场景 | 禁忌行为 | 正确行为 | 版本 |
|:---------|:---------|:---------|:----|
| 发现自己的分析/回答存在逻辑矛盾 | 忽略不处理，继续下一个话题 | 立即记录到 WAL（问题描述 + 上下文 + 触发时间） | v7.1.0 |
| 用户给出明确的行为修正指令 | 口头答应但未固化 | 确认理解后，将修正写入 SELF-EVOLUTION.md 新规则 | v7.1.0 |
| 连续两次犯同类型错误 | 以为会自动修正 | 必须主动发现模式，写入禁止规则 | v7.1.0 |
| 事实性断言被用户指出错误 | 当场道歉但事后不改 | 立即修正输出，同时追加一条规则到零幻觉协议 | v7.1.0 |

## 六、任务反思循环（VOYAGER 模式，来源：ScienceClaw skill-evolution/SKILL.md 适配）

**目的：每次任务完成后强制反思，积累可复用的执行模式，避免重复踩坑。**

### 反思触发条件

| 场景 | 关键任务（编码/搜索/分析/架构决策） | 简单任务（日常问答/闲聊） |
|:-----|:----------------------------------|:-------------------------|
| 是否需要反思 | ✅ 必须 | ⭕ 可选（但不禁止） |
| 反思深度 | 5维度评分 + 模式存储 | 仅记录关键经验 |

### 五维自评标准

| 维度 | 1分 | 3分 | 5分 | 评分锚点 |
|:----|:----|:----|:----|:--------|
| 完整性 | 只做了部分 | 覆盖主体需求 | 覆盖所有边界情况 | 需求文档/对话覆盖度 |
| 准确性 | 有事实错误 | 主要正确 | 零幻觉，可验证链完整 | 工具调用证据链 |
| 效率 | 冗余调用>5次 | 1-3次最优 | 最小工具调用达成目标 | 工具调用次数 |
| 深度 | 表层回答 | 分析了根因 | 多角度验证+根本原因 | 分析层次 |
| 可操作性 | 抽象结论 | 有具体代码/命令 | 用户可直接复制执行 | 产出格式 |

### 反思记录格式（写入 WAL）

每次关键任务完成后，追加一条到 `memory/wal.log`：

```json
{
  "type": "reflection",
  "ts": "2026-05-30T21:30:00+08:00",
  "task": "ScienceClaw 深度分析",
  "domain": "系统演化",
  "scores": {"completeness":4, "accuracy":5, "efficiency":3, "depth":5, "actionability":4},
  "tools": ["gh api", "web_fetch", "read"],
  "pattern": "三轮深入：README→目录结构→核心源文件",
  "lessons": ["GitHub star 核验必须每次都用 gh api 现场查"],
  "improvement_proposal": false
}
```

### 模式进化规则

| 触发条件 | 行为 |
|:---------|:----|
| 同一 domain 出现 3 次 | 检查是否有重复模式，如有 → 写入 SELF-EVOLUTION.md 规则 |
| 同一 task_type 评分>4 连续 5 次 | 提取为可复用技能，更新 skills/ 或 AGENTS.md 经验 |
| 同一 task_type 评分<2 连续 2 次 | 写入禁止模式到禁止用语表，不再用该方式 |
| improvement_proposal=true | 触发离线层 cron 评估是否创建新规则 |

### 执行时机

反思在**本次任务最后一段回复中内联完成**，不占用单独的 cron/定时任务。
